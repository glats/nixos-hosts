# AGENTS.md - NixOS Multi-Host Configuration Repository

## Project Overview

Unified NixOS system and Home Manager configuration for multiple hosts:
- **`"rog"`** — ASUS ROG laptop (x86_64-linux), full desktop with services
- **`"thinkcentre"`** — Lenovo ThinkCentre (x86_64-linux), headless workstation

User `"glats"` on both. Nix Flakes with `lib/mkHost.nix` helper for multi-host support.
Validation via `nixos-rebuild`.

## Repository Structure

```
nixos-hosts/
  flake.nix                      # Flake entry: nixosConfigurations for all hosts
  lib/
    mkHost.nix                   # Helper function to generate host configs
  hosts/
    rog/
      default.nix                # Host-specific: services, nvidia, asus-fan-control
      hardware-configuration.nix # Auto-generated (DO NOT EDIT)
    thinkcentre/
      default.nix                # Host-specific: headless, no services
      hardware-configuration.nix # Auto-generated (DO NOT EDIT)
  modules/
    core/                        # System core: nix, users, packages, zsh, sops, cachix,
                                 #   home-manager, polkit, logind, overlays
    hardware/                    # Hardware: nvidia (rog-only), asus-fan-control (rog-only), keyring
    desktop/                     # Desktop: mate-xrdp, fonts, kmscon, i18n
    services/                    # Services: nginx, jellyfin, gonic, samba, code-server,
                                 #   guacamole, wetty, fileshelter, cobalt, dozzle,
                                 #   ftp, ollama, searx, ddclient, wireguard (rog-only)
    virtualisation/              # Virtualisation: docker, libvirt
    networking/                  # Networking: openssh, avahi, firewall, wol
    home/                        # Home Manager: base, neovim, opencode, shell, tmux,
                                 #   conky (host-specific configs)
  bin/                           # Helper scripts
  pkgs/                          # Custom packages
  docs/                          # Documentation
  secrets/                       # sops-nix secrets (encrypted for all hosts)
```

Multi-host config: `nixosConfigurations."rog"` and `nixosConfigurations."thinkcentre"` with
NixOS-integrated Home Manager. Host-specific modules selected via `hosts/{hostname}/default.nix`.

## Critical Rules

- **Never edit** `hardware-configuration.nix` — auto-generated by `nixos-generate-config`.
- **Category `default.nix`** — each `modules/{category}/default.nix` re-exports all modules
  in that category. Adding a new module requires adding it to the category's `default.nix`.
- **overlays.nix** is NOT a NixOS module — it's imported via `import` in `flake.nix`, not via
  `imports`. It must NOT be added to `modules/core/default.nix`.
- **Home Manager** is NixOS-integrated only. `modules/core/home-manager.nix` is the single
  source of truth for HM module imports.
- **Unfree packages** must be listed in `nixpkgs.config.allowUnfreePackages` or
  `permittedInsecurePackages` in `hosts/rog/default.nix`.
- **Nix experimental features** `nix-command` and `flakes` are enabled system-wide.
- **Flake sources** — `nixpkgs` from `nixos-unstable`, `home-manager` from `master`.
- **All code in English** — no mixed languages. See [docs/style-guide.md](docs/style-guide.md).
- **Formatter** — `nixfmt` (official). Run `format-nix` before finalizing.
